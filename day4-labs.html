Lab 01
Build Your First MCP Tool
Install the MCP SDK, create a local stdio MCP server with two tools — a mock weather tool and an ROI calculator — configure it in Claude Code, and confirm that Claude uses both tools in a real conversation.
⏱ 30 minutes Intermediate Module 1 + 3

Step 1 — Install the SDK and Create the Server File

bash
# Create a project directory
mkdir my-first-mcp && cd my-first-mcp
python3 -m venv .venv && source .venv/bin/activate
pip install "mcp[cli]"

Step 2 — Write the MCP Server

python
"""
server.py — First MCP server with two tools.
Run with: mcp dev server.py
"""
import asyncio, json, math
from mcp.server import Server
from mcp.server.models import InitializationOptions
from mcp.types import Tool, TextContent
import mcp.server.stdio

server = Server("my-first-mcp")

@server.list_tools()
async def list_tools() -> list[Tool]:
    return [
        Tool(
            name="get_weather",
            description="""Get the current weather for a city.
            Use when the user asks about weather conditions, temperature,
            or forecast for a specific location.
            Returns temperature in Celsius and a weather condition description.
            This is a simulated tool for demonstration purposes.""",
            inputSchema={
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "City name (e.g. 'London', 'Tokyo', 'New York')"
                    }
                },
                "required": ["city"]
            }
        ),
        Tool(
            name="calculate_roi",
            description="""Calculate Return on Investment (ROI) for a project or initiative.
            Use when the user wants to compute ROI, evaluate whether a project is
            financially worthwhile, or compare investment options.
            Returns ROI percentage, net profit, and payback period in months.""",
            inputSchema={
                "type": "object",
                "properties": {
                    "initial_investment": {
                        "type": "number",
                        "description": "Total upfront investment in USD"
                    },
                    "annual_benefit": {
                        "type": "number",
                        "description": "Expected annual benefit or revenue in USD"
                    },
                    "years": {
                        "type": "integer",
                        "description": "Evaluation period in years (default: 3)",
                        "default": 3
                    }
                },
                "required": ["initial_investment", "annual_benefit"]
            }
        )
    ]

@server.call_tool()
async def call_tool(name: str, arguments: dict) -> list[TextContent]:
    if name == "get_weather":
        # Mock weather data — in production, call a real weather API
        city = arguments["city"]
        mock_data = {
            "city": city,
            "temperature_celsius": 22,
            "condition": "Partly cloudy",
            "humidity_pct": 65,
            "wind_kph": 18,
            "note": "Simulated data — Lab 1 demonstration"
        }
        return [TextContent(type="text", text=json.dumps(mock_data, indent=2))]

    elif name == "calculate_roi":
        inv = arguments["initial_investment"]
        benefit = arguments["annual_benefit"]
        years = arguments.get("years", 3)
        total_benefit = benefit * years
        net_profit = total_benefit - inv
        roi_pct = (net_profit / inv) * 100
        payback_months = (inv / benefit) * 12 if benefit > 0 else None
        result = {
            "initial_investment_usd": inv,
            "annual_benefit_usd": benefit,
            "evaluation_years": years,
            "total_benefit_usd": round(total_benefit, 2),
            "net_profit_usd": round(net_profit, 2),
            "roi_percent": round(roi_pct, 1),
            "payback_months": round(payback_months, 1) if payback_months else None
        }
        return [TextContent(type="text", text=json.dumps(result, indent=2))]

    raise ValueError(f"Unknown tool: {name}")

async def main():
    async with mcp.server.stdio.stdio_server() as (r, w):
        await server.run(r, w, InitializationOptions(
            server_name="my-first-mcp", server_version="1.0.0",
            capabilities=server.get_capabilities(None, {})
        ))

if __name__ == "__main__": asyncio.run(main())

Lab 02
GitHub MCP Workflow Automation
Configure the official GitHub MCP server with a PAT scoped to your own repositories. Use Claude to list your open pull requests, summarise the most recently changed one, and draft a professional review comment.
⏱ 45 minutes Intermediate Module 2

Step 1 — Create a Fine-Grained GitHub PAT

Go to GitHub → Settings → Developer Settings → Personal Access Tokens → Fine-grained tokens. Create a token with:

  • Resource owner: your account
  • Repository access: Only select repositories (choose 1-2 repos you own)
  • Permissions: Contents: Read, Pull requests: Read and write, Issues: Read and write
  • Expiration: 30 days (this is a lab token)

Store the token: export GITHUB_PAT="github_pat_..." in your shell profile.

Step 2 — Configure GitHub MCP

json
// .claude/settings.json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "${GITHUB_PAT}"
      }
    }
  }
}

Step 3 — Run the Automation Workflow

Open Claude Code and run /mcp to confirm GitHub appears. Then use these prompts in sequence:

PROMPT 1: "List all open pull requests in [your-username]/[your-repo]" Expected: Claude calls list_pull_requests, shows PR numbers, titles, authors PROMPT 2: "Get the details of the most recently updated PR" Expected: Claude calls get_pull_request with the PR number, shows diff stats, review status, CI status, and merge readiness PROMPT 3: "Draft a constructive review comment for this PR. Focus on: 1. What the PR does well 2. Any risks or concerns 3. Suggestions" Expected: Claude generates a professional review comment without calling any tool (it already has the PR context from Prompt 2) PROMPT 4 (optional): "Post that review comment to the PR" Expected: Claude calls add_pull_request_review_comment — confirm before executing
⚠
Review before posting: Prompt 4 will actually post a comment to your GitHub PR. Read Claude's draft carefully before confirming. If you're not comfortable posting to a real repo, skip Prompt 4 and treat the draft as the deliverable.
Deliverables
  • Working GitHub MCP configuration in .claude/settings.json
  • Screenshot or text output showing Claude's PR summary and the drafted review comment
  • One-sentence reflection: what workflow in your current GitHub process could this automate?
Lab 03
Enterprise Security Audit Checklist
Walk through the 15-point production security checklist from Module 4. Assess your (or a hypothetical) Claude deployment against each control. Identify and document your top 3 gaps with remediation plans.
⏱ 30 minutes Advanced Module 4

Scenario

You have just deployed a Claude Code integration for your 15-person engineering team. Claude has access to two MCP servers: the official GitHub server and a custom Python server that can query your PostgreSQL production database. You need to audit this deployment before the company's security team reviews it next week.

✗
Assume the worst: When auditing an AI deployment, assume every gap is actively exploitable. An attacker who can inject a database record with malicious instructions, and your Claude deployment reads that record and has write access to GitHub — that's a full takeover chain. Be ruthless in identifying weaknesses.

Security Audit Worksheet

For each control below, mark it as OK, GAP, or N/A. For each GAP, write the remediation in the space provided.

  • 1. System prompt hardening — Does your system prompt explicitly state that tool results are untrusted data? GAP EXAMPLE
  • 2. Principle of least privilege — Are only the minimum necessary MCP tools allowed? Is the GitHub PAT scoped to specific repos only?
  • 3. Human-in-the-loop for write ops — Does any write operation (GitHub push, DB write) require explicit human approval?
  • 4. Output validation — Is there programmatic validation of Claude's tool call arguments before execution?
  • 5. Input sanitisation — Are database records that enter tool results sanitised before being returned to Claude?
  • 6. Audit logging — Is every tool call logged with user_id, timestamp, tool_name? Stored for at least 90 days?
  • 7. Secrets management — Are all API keys in a secrets manager or shell environment — NOT in committed files?
  • 8. Read-only database — Does the database MCP connect as a read-only user to a replica? GAP EXAMPLE: production primary used
  • 9. Data minimisation — Do database queries select only needed fields? No SELECT * patterns?
  • 10. Cost controls — Are per-session token limits and per-user daily spend limits configured?
  • 11. Rate limiting — Are MCP server endpoints rate-limited per user?
  • 12. Compliance documentation — Are your compliance obligations (GDPR, SOC2) documented and mapped to controls?
  • 13. Network controls — Are MCP servers accessible only via VPN or from specific IP ranges?
  • 14. Incident response plan — Is there a documented plan for what happens if a prompt injection attack succeeds?
  • 15. Regular security review — Is there a scheduled quarterly review of permissions and logged anomalies?

For your top 3 gaps, complete this template:

Step 2 — Build the MCP Server with Auth

python
"""
kb_server_lab.py — Lab 4: KB MCP server backed by JSON file with auth.
"""
import asyncio, json, os
from mcp.server import Server
from mcp.server.models import InitializationOptions
from mcp.types import Tool, TextContent
import mcp.server.stdio

server = Server("acme-kb-lab")

# Load KB data at startup
KB_DATA_PATH = os.environ.get("KB_DATA_PATH", "kb_data.json")
KB_AUTH_TOKEN = os.environ.get("KB_AUTH_TOKEN", "")

with open(KB_DATA_PATH) as f:
    KB_DATA = json.load(f)

def search_articles(query: str, category: str | None, limit: int) -> list[dict]:
    """Simple keyword search over article titles, bodies, and tags."""
    query_lower = query.lower()
    results = []
    for article in KB_DATA["articles"]:
        if category and article["category"] != category:
            continue
        searchable = " ".join([
            article["title"], article["body"], " ".join(article["tags"])
        ]).lower()
        if query_lower in searchable:
            results.append({
                "id": article["id"],
                "title": article["title"],
                "category": article["category"],
                "snippet": article["body"][:150] + "..."
            })
    return results[:limit]


@server.list_tools()
async def list_tools() -> list[Tool]:
    return [
        Tool(name="search_kb", description="Search the knowledge base by keyword query.",
             inputSchema={"type": "object", "properties": {
                 "query": {"type": "string"},
                 "category": {"type": "string"},
                 "max_results": {"type": "integer", "default": 5}
             }, "required": ["query"]}),
        Tool(name="get_article", description="Get full article by ID from search_kb results.",
             inputSchema={"type": "object", "properties": {
                 "article_id": {"type": "string"}
             }, "required": ["article_id"]}),
        Tool(name="list_categories", description="List all KB categories with article counts.",
             inputSchema={"type": "object", "properties": {}})
    ]


@server.call_tool()
async def call_tool(name: str, arguments: dict) -> list[TextContent]:
    if name == "search_kb":
        results = search_articles(
            arguments["query"],
            arguments.get("category"),
            arguments.get("max_results", 5)
        )
        return [TextContent(type="text", text=json.dumps({"results": results}))]
    elif name == "get_article":
        art_id = arguments["article_id"]
        if not art_id.replace("-", "").isalnum():
            return [TextContent(type="text", text=json.dumps({"error": "Invalid article_id"}))]
        article = next((a for a in KB_DATA["articles"] if a["id"] == art_id), None)
        if not article:
            return [TextContent(type="text", text=json.dumps({"error": f"Article {art_id!r} not found"}))]
        return [TextContent(type="text", text=json.dumps(article))]
    elif name == "list_categories":
        return [TextContent(type="text", text=json.dumps(KB_DATA["categories"]))]
    raise ValueError(f"Unknown tool: {name}")

async def main():
    async with mcp.server.stdio.stdio_server() as (r, w):
        await server.run(r, w, InitializationOptions(
            server_name="acme-kb-lab", server_version="1.0.0",
            capabilities=server.get_capabilities(None, {})
        ))

if __name__ == "__main__": asyncio.run(main())

Step 3 — Write 3 pytest Tests

python
"""
tests/test_kb_server.py — Lab 4 test suite. Run: pytest tests/ -v
"""
import json, pytest, asyncio
from mcp.types import TextContent

# Patch env before importing server
import os
os.environ["KB_DATA_PATH"] = "kb_data.json"
from kb_server_lab import call_tool, search_articles


@pytest.mark.asyncio
async def test_search_kb_finds_matching_article():
    """search_kb returns results when query matches article content."""
    result = await call_tool("search_kb", {"query": "deployment"})
    data = json.loads(result[0].text)
    assert len(data["results"]) >= 1
    titles = [r["title"] for r in data["results"]]
    assert any("Deployment" in t for t in titles)


@pytest.mark.asyncio
async def test_get_article_returns_full_body():
    """get_article returns complete article including body and tags."""
    result = await call_tool("get_article", {"article_id": "kb-001"})
    article = json.loads(result[0].text)
    assert article["id"] == "kb-001"
    assert "body" in article
    assert "tags" in article
    assert len(article["body"]) > 0


@pytest.mark.asyncio
async def test_get_article_rejects_invalid_id():
    """get_article returns error JSON (not exception) for invalid article_id."""
    result = await call_tool("get_article", {"article_id": "../../../etc/passwd"})
    data = json.loads(result[0].text)
    assert "error" in data
    # Must not expose the malicious path in the error message
    assert "passwd" not in data.get("error", "")
